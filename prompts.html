プロンプト集（業務で使える例：1〜5）

使い方：文をコピー → Copilot / ChatGPT / Gemini に貼る → 足りない情報だけ追記。
1. 事務・総務 2. 企画・分析 3. IT・開発 4. 営業・顧客 5. 学習・共有

1. 事務・総務

議事録の要約

【目的】会議メモを要点だけにまとめたい 【対象】以下の議事録 【出力】5行の要約＋決定事項＋ToDo（担当・期限） --- （ここに議事録を貼る）
派生例：ビジネス文体化 / ToDoだけ抽出 / 決裁向け1枚化

社内案内メール

【目的】制度変更を社員に案内 【条件】短く・やさしく・見出し付き 【出力】件名案3つ＋本文（注意点・期限・リンクの空欄あり） 【入力】変更点の要点：（ここに列挙）
派生例：対象別に語調変更 / 追伸でFAQリンク追加

2. 企画・分析

データの要点抽出

【目的】表データの要点を抽出 【出力】上昇トレンド上位3つ＋根拠（列名と数値）＋次の打ち手3つ 【入力】表の概要：（列名・期間など） 【データ】（ここに表や数値を貼る）
発展例：経営層向け1枚資料 / ダッシュボード説明文

報告書ドラフト

【目的】週次報告のたたき台を作成 【出力】1. サマリ（3行） 2. 実績 3. 課題 4. 次週計画 5. 依頼事項 【入力】素材：（KPIや出来事を箇条書き）
派生例：部署別版 / クライアント向け版

3. IT・開発

エラー相談（PAC/PADなど）

【目的】エラー原因と対処を知りたい 【環境】（例：Power Automate Desktop / バージョン） 【現象】（エラー文をそのまま貼る） 【再現】（手順を箇条書き） 【試したこと】（実施済みの対策） 【出力】原因候補ベスト3＋確認手順＋回避策
派生例：環境依存を考慮 / 権限・ネットワーク前提を追加

リファクタ相談

【目的】読みやすく安全にしたい 【対象】（関数やフローの抜粋を貼る） 【制約】三項演算子は使わない／コメント多め 【出力】改善方針＋リスク指摘＋サンプル修正（全関数を省略せず提示）
発展例：テスト観点の追加 / コード規約の提案

4. 営業・顧客

顧客メール返信案

【状況】（要望／懸念／期日） 【トーン】丁寧・簡潔・感情を落ち着かせる 【出力】件名案2つ＋本文（結論先出し・箇条書き）＋次のアクション
派生例：社外向けフォーマル化 / 次アクションの明確化

提案書の骨組み

【目的】提案書の構成を作る 【条件】1. 表紙 2. 現状課題 3. 解決案 4. 期待効果 5. 体制 6. スケジュール 7. 価格 8. 次の一手 【前提】（業界／規模／現状） 【出力】各章の見出し＋箇条書き要点
発展例：ROI簡易試算 / 業界別サンプル

5. 学習・共有

ナレッジまとめ

【目的】複数資料から学びを整理 【対象】（資料名やURLの一覧） 【出力】要点5つ＋注意点3つ＋次回アクション3つ＋FAQ候補
派生例：新人向け言い換え / 部署別の観点追加

社内講習スライド草案

【目的】短い勉強会スライドの草案 【時間】15分 【出力】スライド見出し10枚分＋各3行要点＋冒頭の目的1行＋最後のまとめ1行 【対象】（想定受講者）
発展例：想定Q&A追加 / 演習タスク付与
